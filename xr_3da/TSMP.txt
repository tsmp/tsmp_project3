английским на клавиатуре)
-Измененная система реакции движка на ошибки, после некоторых ошибок может продолжить 
работу (например неверный индекс предметов)
-Исправлен вылет при вводе в чат огромного кол-ва сообщений, (все сообщения отрезаются 
до 100 символов)
-sv_no_auth_check возвращена работоспособность команды, по умолчанию включена
-Закрыто большинство ошибок в голосовании приводящих к вылетам (запрет _ и % , ограничение 
до 70 символов)
-Автоматическая замена опасных символов в нике при подключении клиента и смене 
ника (%,_ , >70 символов ник )
-Отключена проверка cd key по умолчанию, можно включить ключом -check_cd_key
-Исправлена ошибка с неработающей сменой карты при голосовании.
-Добавлена возможность в команде sv_kick после имени игрока написать причину кика после 
знака _ , работает ТОЛЬКО для радминов (непонятно почему, для админа не работает...) . 
(пример "ra sv_kick user_za podstvol" , user будет кикнут и получит сообщение "za podstvol") 
обычный кик работает как и раньше.
-Не падает от кривых пакетов (stalkazz), так же автоматически банит ip с которого шла атака
-С ключом -debug вывод отладочных сообщений и информации по пакетам
-Запрещен бан сервером 0.0.0.0 вызывавший вылет.
-С ключом -nospeech запрещает рацию для всех игроков, при попытке 
использования игрок получает сообщение
-исправлен вылет при съедании аптечки когда в тебя стреляют (pIItem->m_pCurrentInventory == this)
-добавлена возможность запретить смену ника, включено по умолчанию,

с ключом -enable_name_change.
-при подключении игроков на ранних этапах выводит их ники в консоль
-исправлена ошибка подключения игроков на сервер с проверкой ключа
- при подключении игроков выполняется команда listplayers
- из sv_listplayers убрано сообщение bad ip adress, которое ничего полезного не сообщало
-исправлены вылеты: (e_parent && e_entity) ; e_entity->ID_Parent == id_parent ;
e_entity->name_replace() ; e_parent->name_replace() ; e_parent ; result <= u32(u16(-1)) ;
Idx<m_items.size() ; Process_update client not found ;  SLS_Clear found;
- Вместо того чтобы крашнуться с указанными выше ошибками сервер продолжает работу и выводит
сообщение с описанием ошибки, пример "! error (e_parent && e_entity)"
- Удалена команда sv_vote_time
- Добавлена команла tsmp_vote_time, отвечающая за время голосования, возможные значения 
15-180 сек, по умолчанию 60
- Теперь сервер выводит не только логин при входе радмина, но и ник, пример : 
# Player [maks0] logged as remote administrator with login [maks].
# srv: Access permitted.
-Добавлено выполнение команд по таймеру, команды прописываются в файл tsmp_timer1_commands.ltx
-Добавлена команда tsmp_timer1_interval, отвечающая за интервал между выполнением команд по таймеру
 по умолчанию 15 мин, возможные значения 1-60
- Добавлен второй таймер, выполняет команды из tsmp_timer2_commands.ltx , поревый из
tsmp_timer1_commands.ltx
- Добавлены команды включения\отключения таймеров tsmp_timer1_enabled и 
tsmp_timer2_enabled возможные значения 0/1.
- Команды интервала таймеров обновлены: tsmp_timer1_interval и tsmp_timer2_interval
значения 1-60 мин.
- Сами таймеры немного оптимизированы, проверка выполняется не каждый раз
- Новая консольная команда tsmp_votesuccess, действует как sv_votestop только наоборот, 
текущее голосование проходит успешно при выполнении данной команды, доступна админам и
радминам.
- Новая консольная команда tsmp_nickname_change_mode, отвечает за режим смены ника на
сервере. 1 - Смена ника разрешена. 2 - смена ника разрешена, если идет голосование - 
запрещена. 3 - запрещена всегда. По умолчанию 3. Менять могут как админы, так и радмины.
Если сервер запущен с ключом -enable_name_change то установится режим 1.
- Добавлена возможность админу писать в консоль большими буквами и двигать место ввода
(курсор правда не двигается :) )
- Добавлен новый ключ запуска -alwaysflush, он переводит движок в новый режим записи лога. Лог
будет записываться на диск при каждом изменении без использования команды flush, если
движок слетит с "прекращена работа программы xr_3da" то лог все равно сохранится, однако 
во время работы сервера усилится нагрузка на диск, могут  возникнуть лаги, так что сами смотрите
надо вам это или нет. По умолчанию отключено.
- В голосовании кое что изменено, должно теперь работать как надо.
- При неудачной попытке входа радмина выводит его ник и логин с которым пытался зайти.
- Добавлена консольная команда tsmp_weapon_disabler_enabled, возможные значения 0/1, 
 команда включает/отключает возможность запрета предметов на сервере.
- Добавлена консольная команда tsmp_weapon_disable для запрета покупки снаряжения,
 команда принимает два аргумента - код предмета в ресурсах игры и 1/0 , 1 включает запрет, 0 
 снимает. Все оружие с 0 прописывать не надо, 0 сделан только для разрешения ранее запрещенного 
 оружия. Пример: tsmp_weapon_disable mp_wpn_rpg7 1 - данная команда запрещает рпг для покупки 
 на сервере.
- Добавлены готовые наборы настроек запрета оружия для удобства админов: granati - запрещает
 покупку всех гранат, podstvol - запрешает покупку всех подствольных гранатометов и патронов к ним,
 hard_weapon - запрещает покупку тяжелого оружия (рпг и бульдог, патроны к рпг). Пример 
 использования:  tsmp_weapon_disable granati 1 - запретит покупку гранат.

 !!! подствол запрещать только командой tsmp_weapon_disable podstvol 1

 - Исправлен баг с невыводом ника игрока который пытался зайти в радминку, теперь всех выводит.
 - Исправлен вылет когда игроки прописывали в голосование на смену карты код приводящий к крашу
 клиентов (пример stalker_story_1), теперь перед голосованием проверяется есть ли такая карта на
 сервере, если нет голосование не начинается и выводится ник игрока который пытался начать 
 голосование, если карта с таким именем есть на сервере, голосование выполняется как и должно.
 - Исправлена ошибка прошлой версии с неверным интервалом второго таймера.
 - Теперь настройки сохраняются и считываются не с user.ltx а tsmp_user.ltx , это сделано
 чтобы не конфликтовали команды патча и оригинального сервера, если они находятся в одной папке.


Список текущих ключей запуска (стандартные не пишу, только новые):
-debug выводит отладочные сообщения и названия пакетов
-check_cd_key включает проверку ключа на сервере, по умолчанию проверка отключена
-test_rus включает русский чат, по умолчанию выключен
-nospeech отключение голосового чата (по умолчанию вызывается клавишам z и c,
 команды В атаку! , купите оружие и т.п.)
-enable_name_change разрешает смену ника игрокам на сервере
-alwaysflush переводит движок в новый режим записи лога, при добавлении нового элемента в 
лог он тут же записыывается на диск, таким образом при любом краше ничего не потеряется,
но может вызывать лаги из за постоянной нагрузки на диск
 
 Консольные команды (доступны админу и радминам):
 -get_server_version - выдает версию и билд сервера, только для админа и радминов;
 -tsmp_timer1_interval - настройка интервала первого таймера выполнения команд, допустимые
  значения 1-60 мин, по умолчанию 15;
 -tsmp_timer2_interval - настройка интервала второго таймера выполнения команд, допустимые
  значения 1-60 мин, по умолчанию 15;
 -tsmp_timer1_enabled - команда отключения/включения первого таймера, значения 0/1, по умолчанию 1;
 -tsmp_timer2_enabled - аналогичная команда для второго таймера, значения 0/1, по умолчанию 0;
 -tsmp_votesuccess - команда для успешного прохождения текущего голосования;
 -tsmp_vote_time - команда позволяет задать время голосования в секундах, возможные значения
 15-180, по умолчанию 60;
 -tsmp_nickname_change_mode - отвечает за режим смены ника на сервере. 1 - Смена ника разрешена. 
 2 - смена ника разрешена если не идет голосование. 3 - запрещена всегда. По умолчанию 3. 
 Если сервер запущен с ключом -enable_name_change то установится режим 1.
 -tsmp_weapon_disabler_enabled - отвечает за включение/отключение возможности запрещать оружие.
 -tsmp_weapon_disable - команда для запрета покупки снаряжения, принимает два аргумента - код 
 предмета в ресурсах игры и 1/0 , 1 включает запрет, 0 снимает. Все оружие с 0 прописывать 
 не надо, 0 сделан только для разрешения ранее запрещенного оружия. 
 Пример: tsmp_weapon_disable mp_wpn_rpg7 1 - данная команда запрещает рпг для покупки на сервере.
 Кроме названия снаряжения команда принимает три готовых набора: granati - запрещает
 покупку всех гранат, podstvol - запрешает покупку всех подствольных гранатометов и патронов к ним,
 hard_weapon - запрещает покупку тяжелого оружия (рпг и бульдог, патроны к рпг). Пример 
 использования:  tsmp_weapon_disable granati 1 - запретит покупку гранат.
